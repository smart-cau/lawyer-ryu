# 06-interview — 변호사 본인 인터뷰 + 사무직원 자료요청

> 본 디렉토리는 **류남경 변호사 대면 인터뷰**와 **사무직원 기초자료 요청**을 분리해서 관리한다.

## 분리 원칙

| 영역 | 대상 | 다루는 내용 |
|------|------|-------------|
| **인터뷰** (변호사) | 변호사 본인 | 메시지 방향성·분야 우선순위·본인 스토리·운영 방침·본사 관계 |
| **자료요청** (사무직원) | 사무직원 | 기존 사진·표창장·자격증·운영시간·도메인·본사 자료 |

메인 인터뷰의 **범위 외**: 페이지 구조 매핑(F/P/S/M) · 구체적 카피 문구 · 사진 신규 촬영 · 언론 보도 노출 · 팀 구성원(단독 운영).
**성공사례 본문 수집은 별도 후속 인터뷰**(`success-cases-interview-guide.md`)로 분리한다.

## 파일 구성

| 파일 | 대상 | 시점 | 용도 |
|------|------|------|------|
| `interview-script.md` | 변호사 | 메인 인터뷰 당일 | 본인 스토리·분야 설계 방향 질문지(받아쓰기 양식 포함) |
| `success-cases-interview-guide.md` | 변호사 | 메인 인터뷰 이후 후속 세션 | 대표 성공사례 라인업 수집 질문지 |
| `office-staff-asset-request.md` | 사무직원 | 인터뷰와 별개로 즉시 송부 | 사진·표창·운영정보·본사 자료 등 기초 요청 |
| `interview-result-YYYY-MM-DD.md` | — | 메인 인터뷰 직후 | 채워진 메인 인터뷰 답변지(파일명에 날짜) |
| `success-cases-result-YYYY-MM-DD.md` | — | 성공사례 인터뷰 직후 | 채워진 성공사례 답변지(파일명에 날짜) |

## 진행 순서

1. **즉시** — `office-staff-asset-request.md` 사무직원에게 송부 (인터뷰와 병행 진행)
2. **메인 인터뷰 당일** — `interview-script.md` 인쇄해서 대면 진행
3. **메인 인터뷰 직후** — `interview-script.md` 사본 → `interview-result-YYYY-MM-DD.md`로 저장
4. **분야 분류 확정** — 메인 인터뷰 Q9·Q10 결과로 최종 분야 라인업 확정
5. **성공사례 후속 인터뷰** — `success-cases-interview-guide.md`의 5부 분야 목록에 4의 결과를 옮긴 뒤 대면 진행
6. **성공사례 인터뷰 직후** — 사본 → `success-cases-result-YYYY-MM-DD.md`로 저장 + 익명화·광고규정 검토
7. **결과 반영** — 영향받는 SoT 문서 일괄 갱신:
   - `docs/03-firm/brand-positioning.md` — 헤드라인 방향·USP 변경분·대표사례 배지
   - `docs/01-profile/profile.md` — 사실관계 보강 (출생연도·공백·수상)
   - 전문분야 페이지 설계서 — 각 분야 "대표사례" 섹션
