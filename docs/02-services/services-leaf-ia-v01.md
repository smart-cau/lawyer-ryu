# Services Leaf IA v0.1

## 1. 1차 페르소나·정서 전제

- **1차 청자**: 외부 검색 직접 진입자. 예 — 네이버·구글에서 "강제추행 변호사", "통매음", "딥페이크 변호사" 검색 → 검색 결과에서 leaf 페이지로 *바로* 진입
- **진입 비중 가정**: services hub 경유보다 외부 검색 직접 진입이 *높다고 가정*. 
- **services hub IA v0.1 §1과의 차이**: hub는 "외부 SEO 진입자 비중 낮음" 가정으로 USP 재노출을 뺐으나, leaf는 정반대 가정. leaf는 *이 페이지가 변호사를 처음 만나는 자리*임을 전제
- **핵심 정서**: 검색 결과 클릭으로 들어온 의뢰인이 5단계 흐름 안에 "내 사건이 여기 있고, 이 변호사가 잘하고, 지금 연락하면 된다"고 확신할 수 있어야 함. 겪고 있는 정서: 억울함, 걱정스러움

## Page IA 구성

### 1. 세부 업무 분야

#### Takeaways

- 이런 분야를 다루는구나
- 내 상황을 쉽게 풀어서 설명해주는 것을 보니 내가 어떤 상황에 처해있는지 이해가 더 잘 된다
- 해당 분야에 대한 변호사의 관점을 보니 전문성이 더욱 신뢰가 간다

#### 콘텐츠 설계 원칙

세부 업무 분야 섹션은 단순한 사건 유형 소개가 아니라, **검색으로 바로 들어온 의뢰인이 자기 사건을 분류하고 수사 초기 대응 방향을 판단하게 돕는 섹션**이어야 한다.

특히 성범죄 leaf처럼 외부 검색 유입 의도가 강한 페이지에서는 "강제추행 변호사", "통매음 고소", "카촬 포렌식", "준강간 혐의", "아청법 처벌", "스토킹 잠정조치" 같은 키워드를 억지로 반복하지 않는다. 대신 각 사건 유형의 실제 쟁점과 초기 대응 문맥 안에 자연스럽게 포함한다.

각 세부 업무 분야는 아래 순서의 정보를 담는 것을 기본값으로 한다.

1. **정의**: 이 사건 유형이 무엇인지 1문장으로 명확히 설명한다.
2. **대표 상황**: 의뢰인이 "내 사건도 여기에 해당하나?"를 판단할 수 있는 생활 장면을 제시한다.
3. **핵심 쟁점**: 수사기관·법원이 실제로 보는 쟁점을 정리한다.
4. **초기 대응**: 경찰 조사 전후에 조심해야 할 행동과 준비해야 할 자료를 안내한다.
5. **변호사 조력 포인트**: 변호사가 필요한 이유를 추상적 전문성 대신 절차·증거·진술·합의 설계 관점으로 설명한다.

#### 현재 구현 기준

현재 `src/app/(frontend)/services/sex-crime/_data/sub-categories.ts`는 세부 업무 분야를 다음 구조로 관리한다.

```ts
type SexCrimeSubCategory = {
  slug: string
  label: string
  keywords: string[]
  definition: string
  body: string
}
```

현 구조에서는 `definition`과 `body`가 하나의 문단으로 이어져 렌더링되므로, 각 필드는 아래 역할을 맡는다.

- `definition`: 사건 유형에 대한 검색 의도의 첫 답변. 굵게 노출되므로 1문장으로 끝낸다.
- `body`: 대표 상황 + 핵심 쟁점 + 초기 대응 + 변호사 조력 포인트를 2~3문장 안에 압축한다.
- `keywords`: 검색어 나열용이 아니라 해당 분류의 범위 확인용이다. 본문에서 모든 키워드를 무리하게 반복하지 않는다.

예시:

```ts
{
  slug: 'assault',
  label: '강제추행·일반 성범죄',
  keywords: ['강제추행'],
  definition:
    '강제추행 사건은 원치 않는 신체 접촉의 경위와 고의성, 피해자 진술의 신빙성이 핵심이 되는 성범죄 사건입니다.',
  body:
    '회식·지인 관계·공공장소·연인 관계에서 발생한 접촉은 당시 분위기보다 수사기관이 보는 증거와 진술 구조가 더 중요하게 작용합니다. 경찰 조사 전에는 접촉 경위, 사후 대화, CCTV·목격자 자료, 합의 가능성을 먼저 정리해야 하며, 성급한 해명이나 직접 연락은 오히려 불리한 정황으로 해석될 수 있습니다.',
}
```

#### 확장 권장 구조

향후 세부 업무 분야 섹션을 더 강한 SEO·전환 섹션으로 만들려면 `definition/body` 2필드 구조보다 아래처럼 의미 단위를 분리하는 편이 좋다.

```ts
type LeafSubCategory = {
  slug: string
  label: string
  keywords: string[]
  definition: string
  commonSituations: string[]
  keyIssues: string[]
  firstResponse: string[]
  attorneyRole: string[]
  relatedKeywords?: string[]
}
```

이 구조로 확장하면 UI는 각 사건 유형을 다음 정보 단위로 노출할 수 있다.

- 제목: 사건 유형 라벨
- 정의: 검색 의도에 대한 짧은 답
- 주로 문제 되는 상황
- 수사에서 다투는 쟁점
- 경찰 조사 전 초기 대응
- 변호사가 개입해야 하는 지점

#### 카피 작성 가드

- "무조건 무혐의", "처벌을 피할 수 있습니다"처럼 결과를 보장하는 표현은 쓰지 않는다.
- 공포 마케팅을 위해 형량·신상정보등록·취업제한만 과도하게 강조하지 않는다.
- 법 조항과 구체 형량은 변경 가능성이 있으므로, 변호사 검수 전에는 단정적으로 길게 쓰지 않는다.
- 모든 사건 유형에 같은 문장 구조를 반복하지 않는다. 사건별로 증거 유형, 절차 리스크, 의뢰인 불안을 다르게 잡는다.
- 변호사 전문성은 "전문적입니다"가 아니라 진술 정리, 증거 선별, 합의 시기, 불기소·기소유예 자료 구성처럼 실제 조력 행위로 설명한다.
