## 결정 (한 줄)

기획·구현 산출물은 성격별로 5개 home(GitHub Issue, ADR, 영역별 자료조사 메모, 코드, 외부 디자인 도구)에 분산한다. 페이지 단위로 산출물을 모으는 폴더(`docs/pages/`)는 *미리* 만들지 않고, 페이지 전용 산출물이 ADR·영역 자료조사 메모 어디에도 안 맞는 형태로 2~3건 쌓이는 시점에 lazy 신설한다.

> **2026-05-13 용어 정리 노트**: 본 ADR 작성 당시 표현 "영역별 SoT"는 `docs/01-profile/`·`docs/03-firm/` 등 영역 디렉토리를 **권위 있는 단일 진실 원천**으로 가정한 것이지만, 이후 합의에서 `docs/` 하위 문서는 **자료조사 메모**일 뿐 카피·IA의 권위 있는 원천이 아님으로 정리되었다. ADR 결정 본문의 *룰 자체*(산출물 5종 home 분산, lazy 신설 트리거)는 그대로 유효하며, 용어만 "영역별 자료조사 메모"로 갱신한다.

## 맥락

기존 `docs/`는 출처별 자료조사 + 일부 가공물(IA·스키마) + 메타로 구성되어 있어 페이지 구조(main/about/services/cases)와 1:1이 아니다. "한 페이지의 기획 결과를 한 폴더에 모으겠다"는 직관은 자연스럽지만, 이 직관을 따르면 두 가지 문제가 발생한다:

1. **중복**: 한 페이지(예: 랜딩)는 여러 영역 자료조사 메모(`03-firm` + `01-profile` + `02-services`)를 인용한다. 페이지 폴더에 *복제*하면 메모가 분산되어 stale 위험이 커진다.
2. **빈 폴더**: 1차 시안 시점에는 페이지 단위로 길게 쌓일 산출물이 거의 없다 — ADR·영역 자료조사 메모·코드·디자인이 대부분 흡수. 미리 만들면 빈 폴더가 된다.

`docs/`의 디렉토리 성격은 README의 "기존 docs/ 디렉토리의 성격" 표 참조.

## 옵션 검토

- A. 모든 페이지 산출물을 별도 폴더(`docs/pages/{main,about,services,cases}/`)에 *미리* 만들고 모은다.
- B. **성격별 home 5개에 분산. `docs/pages/`는 lazy 신설 ← 선택**
- C. 기존 `docs/`를 페이지별로 재구조화 (`01-profile`→`about/`, `02-services`→`services/` 등).

B 채택. 이유:
- 산출물의 80%는 이미 home이 있다(ADR, 영역 자료조사 메모, 코드, 외부 디자인 도구). 별도 폴더는 *진짜 페이지 전용 산출물*이 쌓일 때만 필요.
- A는 빈 폴더·중복 위험. 미리 폴더를 만들면 *그곳에 뭐든 쓰려는 유혹*이 생겨 영역 자료조사 메모가 비어가고 페이지 폴더에 카피가 박혀 재사용 깨짐.
- C는 1회성 비용 큼. `03-firm`처럼 main과 about/firm 양쪽인 디렉토리는 분할이 어렵고, 출처(누가 한 말인지)를 페이지 단위로 흩으면 자료조사 무결성이 흔들린다.

## 결과

- `docs/07-development/INDEX.md`에 산출물 매핑 룰을 박아둠 (산출물 5종 → 위치 표, 자주 헷갈리는 케이스).
- 기존 `docs/` 디렉토리 8개의 성격을 INDEX 표로 정리.
- lazy 신설 시 정확한 구조 명시: `docs/pages/{main,about,services,cases}/`.
- lazy 신설 트리거: 페이지 전용 산출물이 ADR·영역 자료조사 메모에 안 맞는 형태로 2~3건 쌓일 때.

## 후속 트리거 (재검토 시점)

- 페이지 전용 산출물이 임계값(2~3건)을 넘기면 → `docs/pages/{main,about,services,cases}/` 신설하고 신설 자체를 결정 이력에 기록 (이 ADR을 supersede하지 않음 — 룰의 *적용*이지 *변경*이 아님).
- 영역 자료조사 메모가 페이지 구조와 정합이 맞는 형태로 자연 변형되면 → 매핑 룰 재검토 (재구조화 비용 vs 정합성 이득). 이 경우 본 ADR을 supersede.
- 페이지 폴더가 생긴 후 *재사용 가능한 콘텐츠*가 페이지 폴더에 박히는 패턴이 보이면 → 매핑 룰의 강도 재검토 (가이드를 더 강하게 박을지).
