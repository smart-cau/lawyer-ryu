## 0. 이 가이드의 범위

이 문서는 **lawyer-ryu 프로젝트(법무법인 인유 창원사무소 / 류남경 대표변호사 홈페이지)** 카피 작업에 *특화된* 도메인 어휘 가이드입니다. 일반 한국어 humanizer 패턴 1~41과 *직교*하는 별도 검사 축이며, 이 프로젝트의 모든 카피·메타·CTA·footer 작업에서 점검합니다.

다른 도메인(의료·학술·일반 비즈니스)은 이 프로젝트 범위 밖이라 다루지 않습니다. 필요해지면 별도 파일로 분리하세요.

## 4. 법리 검증 체크리스트 (모든 법률 카피 문장에 강제 점검)

광고규정 가드(§3)는 *마케팅 표현 규제*를 본다면, 법리 검증은 *법학 명제로서의 정확성*을 봅니다. 둘은 다른 축이며 둘 다 통과해야 합니다.

법률 카피 문장 단위로 다음 6항을 점검합니다. **위반 시 해당 문장을 수정·삭제·완화한 뒤 재점검**합니다.

### 4.1 법리 정확성 — 형법·형사소송법 기본 원리에 부합하는가

- 범죄성립 3요소(구성요건·위법성·책임) 중 하나만 언급하면서 *전부인 양* 단정하지 않는가
- 공소시효, 미수·기수, 정범·공범, 위법성 조각, 책임 조각 등의 개념을 *일면적으로* 단순화하지 않는가
- 절차법 진술이 실제 형사소송 단계(입건·수사·송치·기소·공판·항소·상고)와 정합한가
- 변호인·변호사·법무법인의 권한 범위를 *과대* 진술하지 않는가 (예: "법원의 판단을 좌우합니다" X)

**예시 (검출 → 수정)**:
- ❌ "형사 범죄는 법이 정한 구성요건이 모두 충족돼야 성립합니다." → 위법성·책임 누락
- ✅ "형사 범죄가 성립하려면 법이 정한 요건을 빠짐없이 갖춰야 합니다." (3요소 함의를 모호한 표현으로 흡수)

### 4.2 단정 표현 회피

- "반드시", "무조건", "한 X라도 흠결되면 Y" 같은 단정 형식
- 변호사 자신의 행위를 *결정적*인 양 진술하는 어휘
- 사실 진술인 척하지만 *항상 그렇다*는 함의를 강하게 만드는 형식

**예시**:
- ❌ "한 요소라도 흠결되면 범죄는 성립하지 않습니다." → 흠결 여부 자체가 해석 영역
- ✅ "사건의 요건을 항목별로 검토해 다툴 여지가 있는 지점을 찾습니다." (행위 진술 + 해석 가능성 인정)

### 4.3 결과 함의 검토

표면은 사실 진술이지만 *결과 보장 인상*을 주는 표현을 검출합니다. 광고규정 §1 #1 (제4조 ③·⑬)와 인접한 회색 지대.

- "반박하기 어려운 / 반박할 수 없는" → 반박 가능성을 변호사가 단정
- "결정적인 / 확실한 / 명백한" → 검사·법원 판단 영역을 변호사가 미리 단정
- "만들어 냅니다 / 이끌어 냅니다 / 가져옵니다" (결과를) → 결과 도출 주체 단정
- "수사기관도 인정할 수밖에 없는" → 상대방 반응을 단정

### 4.4 실무 정합 — 묘사된 절차·행위가 실제 형사 변호 실무와 일치하는가

- "한 줄 의견서" → 의견서는 통상 다층 쟁점·다수 페이지로 구성. 비유로는 OK이나 *행위 진술*에서는 정확성 잃음
- "직접 수사" → 변호사는 *수사기관*이 아님. "직접 조사·사실 확인·증거 수집" 정도가 정확. USP 1의 "검사처럼 수사한다"는 *방식 비유*로 수용되나, 본문 동사 진술에서는 절차 어휘로 분리 필요
- "법원에 출두" / "검찰에 신청" 등 절차 어휘는 실무 명칭과 일치하는지 확인

### 4.5 과장·과소 표현

- 미세 과장: "한 점이면 충분합니다", "한 줄이면 끝입니다" — 강조 효과는 있으나 *법리적 정확성* 측면에서 과장
- 미세 과소: "그저 도와드립니다", "단순한 자문에 그칩니다" — 변호사 권한·역할 과소 진술
- 양극단 모두 의뢰인 오해 유발

### 4.6 의뢰인 오해 가능성

일반 의뢰인이 카피를 읽고 *잘못된 기대*를 형성할 표현인가:

- "수사를 직접 한다"는 표현을 의뢰인이 *변호사가 수사기관 권한을 가진다*로 오해할 수 있는가 → 본문에서 사실 확인·증거 수집 등 평이 어휘로 부연
- "재판 전 종결" 표현을 의뢰인이 *항상 가능*으로 오해할 수 있는가 → "사건마다 다릅니다" 가드 필수
- "구성요건 흠결 = 무죄" 단순화로 의뢰인이 *모든 사건에 적용*된다고 오해할 수 있는가 → "다툴 여지가 있는 지점" 식의 절차 진술로 대체

### 검증 적용 시점

- **카피 후보 작성 단계**: 후보 표 작성과 동시에 6항 자가 점검
- **카피 다듬기·humanizer 호출 시**: 자연스러움 다듬기와 *별도 패스*로 법리 검증 실행
- **카피 코드 반영 직전**: PR 또는 commit 직전 마지막 점검
- **광고규정 가드(§3)와 항상 동시 적용**: 두 검사는 다른 축

## 6. 화자별 톤 가이드

### 6.1 변호사 화자 — M-02 USP, About 의 일부

- 격식체 `~합니다` 종결
- **행동 서술**이 기본 (USP heading은 변호사가 *무엇을 하는지* 동사로)
- 1인칭 "저" 노출은 신중 — 사무소·변호사 일반 주어로
- 자기 평가·과시 톤 회피 (실력·노력 자랑은 hype)
- 광고규정 §1 #4 (연고) 항상 의식 — 검사 19년은 *지식·기술·노하우*로만

### 6.2 의뢰인 화자 — M-05 Process

- 격식체 `~합니다`이지만 *의뢰인이 어떤 단계를 거치는지* 시점
- 동사 주어가 의뢰인 또는 사건 — "변호사가 입회합니다"가 아니라 "조사 자리에 변호사가 함께합니다"
- 안내·동행 톤 — 압박·재촉 X

### 6.3 사무소 화자 — M-07 Contact, Footer

- 정보 전달 위주 — 평이한 격식체
- 마케팅 어휘 X (이미 도달한 청자에게 무슨 말 더 하지 말 것)
- "광고책임변호사 류남경" 등 규정 표기 누락 점검 (§0 #18)

## 8. 카피 길이 정책

분량 정책은 *균질·타이트* 관리가 아니라 *여유·자연 우선*입니다. 글자 수를 빡빡하게 맞추면 비문·억지 어순·생략이 생깁니다. 모바일 가독성 상한만 강제하고, 그 안에서 자연스러움을 우선합니다.

### body 1개 (heading 아래 본문 단락)

| 항목 | 값 |
|---|---|
| 권장 범위 | **90~200자** |
| 절대 상한 | **240자** (이상은 분할 또는 다른 슬롯으로 이전) |
| 절대 하한 | **60자** (이하는 메시지 부족 의심) |
| 문장 수 | **2~4문장** (3문장 고집 X) |
| 균질성 | **강요 X** — 카드별 메시지 밀도 차이 인정 |

### 한 문장

| 항목 | 값 |
|---|---|
| 권장 범위 | **30~80자** |
| 절대 상한 | **90자** |
| 절대 하한 | **20자** (이하는 단조·끊김) |

### 모바일 가독성 기준

- 모바일 360px 폭, 본문 16px, 한 줄 약 **17~22자** (한국어 기준)
- 한 문장 60자 → 약 3줄
- 한 문장 80자 → 약 4줄 (가독성 상한)
- 한 문장 100자 → 약 5줄 — **지문화 시작**, 회피
- 한 문장 90자 초과는 *반드시* 분할 검토

### 문장 복잡도 (글자 수만으로 못 잡는 영역)

글자 수가 권장 안에 있어도 다음 패턴은 분할·어순 단순화 검토:

- **의문절 1개까지 권장** — 한 문장 안에 `~인지/~지/~할지/~느냐` 의문절이 2개 이상이면 호흡 막힘
  - ❌ "어떤 사유가 영장 기각으로 이어지는지, 강제수사 시점에 의뢰인이 가져야 할 권리가 무엇인지 정리합니다." (의문절 2개)
  - ✅ "영장 기각 사유와 강제수사 시점의 권리를 정리합니다." (명사구 결합)
- **명사화 체인 3개 이상 결합 시 검토** — `N의 N의 N`, `~하는 N의 ~하는 N` 등 명사 적층은 humanizer 패턴 11(명사 과다) 약한 신호
  - ❌ "검사가 어떤 순서로 어디를 먼저 보는지 직접 경험한 시점에서 의견서의 구조와 표현을 정리합니다." (명사화 3중)
  - ✅ "검사가 의견서의 어디를 먼저 보는지 알기 때문에, 그 시점에서 구조와 표현을 정리합니다." (인과 동사화)
- **수식어 위치 단순화** — 후치 부사구(`~에 비춰` 등)와 의문절이 한 문장에 결합되면 분할 권장

### 카드 간 상대 분량 (균질성 강요는 X, 비대칭만 점검)

- 균질성 강요는 X — 카드별 메시지 밀도 차이 인정
- 단 **한 카드가 다른 카드의 1.5배 이상**이면 검토 (UI 카드 높이 시각 비대칭, 그리드 레이아웃 깨짐)
- 검토 결과:
  - 메시지 밀도 차이가 정당하면 유지
  - 분량 줄일 여지 있으면 축소
- **계산 기준**: 가장 짧은 카드 × 1.5 ≥ 가장 긴 카드 — 충족하면 통과

### Hero 등 함축형 슬롯의 예외

- Hero 헤드라인·overline·CTA는 분량 정책 적용 X (별도 함축형 룰)
- 본 정책은 *body·설명형 카피*에 적용 (USP body, Process 단계 설명, Services leaf 설명, About 본문 등)

### 점검 시점

- 카피 후보 작성 시 글자 수 자가 측정 (3문장 * 평균 50자 = 150자가 1차 기준값)
- humanizer 호출 시 자동 점검 — 위반 시 출력의 "도메인 어휘 점검" 다음에 "**카피 길이 점검**" 라인으로 명시
- 위반 시 메시지 분할·분량 축소·다른 슬롯 이전 중 선택

## 9. 잠재 의뢰인 심리 컨텍스트

형사 피의자·피고인은 다음 정서 상태에서 페이지에 도달합니다.

- **충격**: 갑작스런 입건·소환·압수
- **불안**: 결과 예측 불가, 정보 비대칭
- **수치심**: 사건 자체에 대한 사회적 시선 (특히 성범죄)
- **억울함**: 자신의 진술과 수사 진술의 격차
- **시급성**: 첫 조사 임박, 시간 압박

**카피 적정선**:
- ❌ 과한 경고 톤 ("이대로면 큰일납니다", "지금 당장 하지 않으면…") — 자극·품위 위반(§0 #20) + 청자 패닉화
- ❌ 과한 안심 톤 ("문제없습니다", "걱정 마세요") — 결과 보장 인상 + 신뢰 손상
- ✅ 사실 + 경로 제시 ("첫 조사부터 단계마다 동행합니다", "가능한 형태는 사건마다 다릅니다")

## 10. lawyer-ryu 특화 결정 (자주 잊는 가드)

이 프로젝트에서 반복적으로 잊거나 위반하기 쉬운 결정. 카피 작업 직전 1순위 점검:

1. **결과 약속 톤 회피** — "끝낸다"는 회피
2. **heading 종결어미** — `~한다` 평서체 X. **변호사 행동 서술 + `~합니다`** 격식체로 본문과 톤 일관 (USP 카드 4장, 서비스 leaf 카드 등)
3. **'A는 B가 아니라 C' 안티패턴 회피** — 부정+대안 구조 (X 대신 Y / X에 그치지 않고 Y / X가 아닌 Y) 카피 클리셰. 단언·시간 진술·행동 서술로 풀어쓸 것 (humanizer 패턴 41)

## 11. 출력 형식 통합

humanizer가 lawyer-ryu 카피를 분석할 때, 출력에 **도메인 적합성** 섹션을 추가합니다.

```
## 분석 결과

**감지된 패턴 요약**: S1 N건, S2 M건, S3 K건
**도메인 적합성** (lawyer-ryu 법률 카피): 부적합 어휘 X건 / 광고규정 가드 위반 위험 Y건 / 법리 검증 위반 Z건
**자연도 등급**: A | B | C | D — [등급 부여 근거 한 줄]

[패턴 분석 ...]

**도메인 어휘 점검**:
- "[회피 어휘]" → [문제 유형 — 비유체/외래어/비즈니스 톤/광고 cliche/광고규정 §X #Y] → "[대체 어휘]" 권장
- ...

**광고규정 교차 점검** (위반 위험이 있을 경우만):
- "[해당 표현]" → 광고규정 §X #Y 위반 위험 → 회피 또는 [대체 표현] 권장

**법리 검증 점검** (§4 위반 시 — 모든 법률 카피에서 강제 점검):
- "[해당 문장]" → §4.1 법리 정확성 / §4.2 단정 / §4.3 결과 함의 / §4.4 실무 정합 / §4.5 과장 / §4.6 의뢰인 오해 중 어느 항목인지 명시 → [수정 방향]

**카피 길이 점검** (§8 위반 시 — body·문장 단위 측정):
- body N자 — 권장 90~200자 / 상한 240자 (위반 시 분할 또는 다른 슬롯 이전)
- 문장 N자 — 권장 30~80자 / 상한 90자 (위반 시 분할)
- 위반 없으면 한 줄 통과 표기 ("body N자·최장 문장 N자 — 정책 통과")
```

도메인 어휘 점검·광고규정 교차·법리 검증·카피 길이는 모두 패턴 1~41과 별도 축이므로, 패턴 카운트와 합산하지 않고 별도 표기.
